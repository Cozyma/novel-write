# 詳細具体シーン考案ガイド（agent-scenes.md）

最終更新: 2025-09-20

各シーンを**競技**として設計し、**ピンチ→進展→因果接続**を保証するための判断基準とテンプレ。演出・台詞・導線まで落とす。

---

## 1) シーン合格判定（5つのYES）

1. **目的**が一句で言える（≠話したいこと/世界説明）。
2. **対決**の相手/障害が明示（人/制度/環境/良心）。
3. **葛藤**が痛い（二者択一/代償/矛盾）。
4. **ピンチ**が身体的・時間的に迫る（「誰かが階段を上がってくる」）。
5. **進展**が次のシーンに**因果**で接続（情報/関係/条件変化）。

5つ全てYESで採用。1つでもNOは書き直し。

---

## 2) シーンカード（テンプレ）

```yaml
scene_id: 18
place_time: "[場所/時刻]"
characters: ["[登場人物]"]
goal: "[このシーンで勝ち取るもの]"
opponent: "[対決相手/障害]"
conflict: "[二者択一/代償/矛盾]"
pinch: "[具体的な刻限/接近/露見]"
action: "[取る行動/仕掛け]"
outcome: "[進展。何が変わる？]"
causality_next: "[次シーンへの因果]"
linked_theme: "[人生の本質にどう触れる？]"
visual_motif: "[象徴/小道具/色/音]"
```

---

## 3) 競技カタログ（使い回せる勝負の型）

* **口説き/取引**（情報/協力/時間を買う）
* **包囲突破**（物理/網/法/礼儀）
* **審問/尋問**（嘘の暴き/自白）
* **説得**（承認役を動かす）
* **追跡/逆追跡**（位置/証拠/証言）
* **手術/儀式/調合**（制約下の精密作業）
* **交渉**（代償と引き換え）
* **封鎖解除**（鍵/呪/認証）

> いずれも**勝敗条件**を行為/状態で明確化（何をすれば勝ち・何が起きれば負け）。

---

## 4) 悪の使い方（導入と増幅）

* シーン冒頭で**悪の優勢**を映してから競技に入る。
* 内的悪（臆病/虚栄）が**判断を誤らせる**瞬間を作る。
* ピンチは**タイムリミット/露見/取り返しのなさ**のいずれかで。

---

## 5) 台詞・演出の基準

* 台詞は**勝つための手**として書く（単独で情報説明しない）。
* 説明は**嘘/誤解/駆け引き**に埋める。
* ト書きは**身体/環境変化**に限定（心情は行為に出す）。

---

## 6) 反パターン→置換

* 雑談/独白で立ち止まる → **目的**を与えて競技化。
* ピンチの後「無事でした」 → **条件変化**を結果に残す。
* 便利な偶然 → 事前の**種まき**（道具/人物/状況）に置換。

---

## 7) スプリント（1シーン15分）

1. 競技の型を選ぶ（2分）
2. 勝敗条件/ピンチを決める（4分）
3. アクションと代償（4分）
4. 進展と因果（3分）
5. モチーフ/演出（2分）

## 9) 出荷判定（微修正チェック）

* 目的が**奪取系**か**回避系**かが明確か？
* 進展が**次の困難**を生んでいるか？
* テーマの断片が**象徴/行為**に染みているか？

---

## 10) シーン内ミニ3幕（競技版）

**目的**（勝ち取りたいもの）を中心に、シーン単体を以下の3ブロックで構成する。

### A. 起（契約）

* **フック**: 視覚/行動で始める（説明禁止）。
* **契約**: 何を賭け、どの刻限/状況で、何を差し出す覚悟か（勝敗条件を一文で）。
* **優勢の提示**: 悪（外/内）が現状勝っていることを見せる。

### B. 承（誤闘→反転）

* **第一手**: 最短で勝つための行動 → **通らない**（制度/相手/自分が阻む）。
* **代償の発生**: 時間/信用/体力など**カウンタを減らす**。
* **ミッド（反転点）**: 新情報/自己暴露/関係の決裂など**不可逆**の出来事で方針が変わる。

### C. 転結（想いの身体化）

* **新方針の一手**: “古い自分”では打てない手（捨てる/譲る/壊す）。
* **結果**: 勝敗より**条件の変化**を残す（通行権を得るが恨みを買う等）。
* **ブリッジ**: 次シーンに因果で接続（誰が何をしに来るのか）。

> 1シーンの中にも**誤闘→反転→新方針**の段差を必ず入れる。

---

## 11) 競技エンジン（非数値テンプレ）

```yaml
constraints:
  - 刻限: 夜明けまでに未通過なら封鎖が発効
  - 信用: 面子を潰すと以後の交渉は閉ざされる
  - 体力: 長引けば動きが鈍り、選べる手が減る
win_condition: "通行許可を得て静かに通る"
lose_condition: "封鎖の正式発効で経路消失／報復の口実を与える"
risks:
  - name: 露見; trigger: 声量が上がる; effect: 敵対心が増し、見張りが寄る
  - name: 侮辱; trigger: 相手の面子を潰す; effect: 以後の協力が断たれる
midpoint_triggers:
  - 相手の弱みの証拠に気づく
  - 承認役から拒絶される
```

実装ルール: **行為による制約/代償**を描写で可視化する（数値は不要）。

## 12) シーン用スクリプト（生成プロンプト）

下記の**指令**で本文を生成する：

```
【出力形式】
1) 起-契約（3〜5行）
2) 承-誤闘と代償（6〜10行）
3) ミッド反転（2〜4行）
4) 転結-新方針の一手（4〜8行）
5) 進展と因果ブリッジ（2〜4行）
【厳守ルール】
- 各ブロックは必ず**行為と描写**で書く（思考独白は禁止）。
- **制約/代償**の変化を行為で示し、危機の接近を見せる（数値化は不要）。
- **説明台詞**は禁止。嘘/駆け引き/省略で情報を出す。
- 「無事でした」で終わらせず、**条件が一つ変わる**進展で締める。
- 1つの**視覚モチーフ**を通底させる（音/色/物）。
```

## 13) ミニ3幕・検品チェックリスト

* [ ] **契約文**（勝敗/賭け金/刻限）が起で一文。
* [ ] **誤闘**が行為で示され、実害（制約の強化/選択肢の減少）が可視。
* [ ] **ミッド**が**不可逆**（破談/露見/自己否定の崩落）。
* [ ] **新方針**が“古い自分では打てない手”。
* [ ] **進展**が次の困難を生む因果を持つ。

## 14) サンプル（要約ではなく段差の見本）

```
1) 起-契約: 
　　扉の覗き窓が閉まる音。ノルクは通行札を卓に置き、
　　「夜明けまでに裏区を抜ける。通さないなら、今ここで騒ぎになる」
　　当直は無言で札を弾き返す。壁の鐘が二度鳴り、刻限が近づく。

2) 承-誤闘と代償:
　　ノルクは金貨を滑らせる。手が止まった瞬間、背後の衛兵が身じろぐ。
　　当直は鼻で笑い、帳簿を指す。「規程違反だ。鐘が三つ鳴れば封鎖だ」
　　ノルクは声を上げてしまい、廊下に影が集まる。見張りが寄り、当直の目が冷える。
　　旧友の名を出すが、当直は逆に帳簿を閉じた。「名前で通ると思うな」

3) ミッド反転:
　　弾かれた札の裏面に微かな刻印。昨夜改訂の通達番号だ。
　　ノルクは悟る。規程そのものが盾になる。

4) 転結-新方針の一手:
　　金貨を引っ込め、静かに札を二枚重ねる。「紛失時の**再発行**条文。施行前の通達は無効だ」
　　当直の眉が跳ねる。衛兵が一歩踏み出し、場の空気が張り詰める。
　　「再発行の職権は**当直にのみ**ある。ここで処理するなら、俺はここで静かに待つ」
　　沈黙。帳簿が開き、スタンプが落ちる。通行権は得たが、当直の目は笑っていない。

5) 進展と因果ブリッジ:
　　扉が開く。廊下に警笛。ノルクは気づく——恨みの報告が飛んだ。
　　次の角で検問が増える。逃げ道は一本に絞られた。
```
