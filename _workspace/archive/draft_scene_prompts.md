# シーン生成プロンプト（ドラフト）

以下は旧 `agent_scenes.md` から抽出した、シーン執筆において非常に有用なプロンプトおよびガイドラインのドラフトです。今後、正式なシステムプロンプトへ組み込むための素材として使用します。

## 1. シーン合格判定（5つのYES）
シーンを生成・検証する際は以下の5条件を必ず満たすこと。
1. **目的**が一句で言える（≠話したいこと/世界説明）。
2. **対決**の相手/障害が明示（人/制度/環境/良心）。
3. **葛藤**が痛い（二者択一/代償/矛盾）。
4. **ピンチ**が身体的・時間的に迫る（「誰かが階段を上がってくる」）。
5. **進展**が次のシーンに**因果**で接続（情報/関係/条件変化）。

## 2. ミニ3幕（競技版）
**目的**（勝ち取りたいもの）を中心に、シーン単体を以下の3ブロックで構成する。
- **A. 起（契約）**
  - **フック**: 視覚/行動で始める（説明禁止）。
  - **契約**: 何を賭け、どの刻限/状況で、何を差し出す覚悟か（勝敗条件を一文で）。
  - **優勢の提示**: 悪（外/内）が現状勝っていることを見せる。
- **B. 承（誤闘→反転）**
  - **第一手**: 最短で勝つための行動 → **通らない**（制度/相手/自分が阻む）。
  - **代償の発生**: 時間/信用/体力など**カウンタを減らす**。
  - **ミッド（反転点）**: 新情報/自己暴露/関係の決裂など**不可逆**の出来事で方針が変わる。
- **C. 転結（想いの身体化）**
  - **新方針の一手**: “古い自分”では打てない手（捨てる/譲る/壊す）。
  - **結果**: 勝敗より**条件の変化**を残す。
  - **ブリッジ**: 次シーンに因果で接続（誰が何をしに来るのか）。

## 3. 競技カタログ（使い回せる勝負の型）
各シーンをいずれかの「勝負の型」に当てはめ、勝敗条件を行為/状態で明確化する。
- **口説き/取引**（情報/協力/時間を買う）
- **包囲突破**（物理/網/法/礼儀）
- **審問/尋問**（嘘の暴き/自白）
- **説得**（承認役を動かす）
- **追跡/逆追跡**（位置/証拠/証言）
- **手術/儀式/調合**（制約下の精密作業）
- **交渉**（代償と引き換え）
- **封鎖解除**（鍵/呪/認証）

## 4. 本文生成用スクリプト（プロンプト指示）
本文を生成する際は、以下の指令を厳守する。

```text
【出力形式】
1) 起-契約（3〜5行）
2) 承-誤闘と代償（6〜10行）
3) ミッド反転（2〜4行）
4) 転結-新方針の一手（4〜8行）
5) 進展と因果ブリッジ（2〜4行）

【厳守ルール】
- 各ブロックは必ず**行為と描写**で書く（思考独白は禁止）。
- **制約/代償**の変化を行為で示し、危機の接近を見せる（数値化は不要）。
- **説明台詞**は禁止。嘘/駆け引き/省略で情報を出す。
- 「無事でした」で終わらせず、**条件が一つ変わる**進展で締める。
- 1つの**視覚モチーフ**を通底させる（音/色/物）。
```
