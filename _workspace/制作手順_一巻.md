一巻（プロローグ〜第五章）作成手順（再現用資料）

目的
- 本リポジトリ内の一巻分ドキュメント（本文・プロット・設定）を、現在の構成と文体方針で再現できるよう、手順を自然言語で記録する。

最終成果（構成の要点）
- 文体方針: 「ノルク＝簡潔＋最小の配慮＋時折の乾いた皮肉」「視点＝ノルクの観察・推定を基調。観測根拠がある断定は可」
- 章立て: プロローグ＋第一章〜第五章、各章5シーン（本文・プロット各1ファイル）
- 命名・並び順: VSCodeで自然に話数順に並ぶよう数値プレフィックス（例: 本文/01-03_街道の小競り合い.txt、プロット/01-03_街道の小競り合い_構成.txt）
- フォーマット: すべて .txt（.md は削除済み）

リポジトリ構成（抜粋）
- docs/
  - 本文/（00_プロローグ.txt、01-01_…〜05、…、05-05_巻の余韻.txt）
  - プロット/（00_プロローグ_構成.txt、01-01_…_構成〜05、…、05-05_巻の余韻_構成.txt、第一巻プロット.txt、執筆方針とTODO.txt）
  - 設定/（コンセプト.txt、文体ガイド_ノルクの口調.txt ほか）

手順の概観
1) 文体・視点ガイドの整備と、既存本文の口調調整
2) マーレンの初登場・初戦闘の演出強化（にじみ＋示唆の方針）
3) 第一章シーン⑤の追加（余韻と小さな変化）
4) 第二章（01〜05）プロット＋本文の追加
5) 第三〜第五章（各01〜05）プロット＋本文の追加（巻末まで）
6) 目次と巻プロットの更新
7) .md → .txt への複製、.md の削除
8) VSCodeでの自然ソート用に数値プレフィックスへ改名

詳細手順
1. 文体・視点ガイドの追加・更新
   - 追加: docs/設定/文体ガイド_ノルクの口調.txt
     - ルール: 簡潔＋最小の配慮／一語の言い捨て抑制／皮肉は各シーン1回程度
     - 視点・叙述: 観測根拠がある断定は可、根拠が薄い場合は示唆（〜に見える など）
     - サンプル置換とチェックリストを記載
   - 反映: docs/設定/コンセプト.txt、docs/プロット/執筆方針とTODO.txt に参照追記

2. 既存本文の口調調整（ノルク）
   - 対象: 第一章_01〜04（言い捨ての削減・最小のクッション付与、例: 「内容」→「内容は？」、「沈黙」→「以後、黙っていてくれ」）
   - 「結果も書け」→「記してくれ。併せて“結果”も頼む」など、配慮の一言を統一

3. マーレンの演出補強（示唆ベース）
   - 初登場（第一章_02）: 姿勢・目配り・清潔感で生まれと鍛錬を「滲ませる」一文を挿入
   - 初戦闘（第一章_03）: 盾落ちカバー（半歩割り込み→角度殺し→肩で重心戻し）。「出力の立ち上がりが卓越している」で常人超えを示唆

4. 第一章の余韻追加
   - 追加: docs/本文/01-05_余韻と小さな変化.txt、対応プロット
   - 目的: 相互評価（型の綺麗さ／非殺の合理）と次章の赤線導線

5. 第二章の作成（01〜05）
   - プロット作成 → 本文執筆の順で追加
   - 主な要素: 酸の層の追跡／巣穴の外観察／配り手の非殺捕捉／隊への共有／赤線の延長

6. 第三章の作成（01〜05）
   - 合同踏査の許可→外周制圧→中枢封止→逮捕と火の手→帳場の余韻
   - ノルクの「外殻だけ切る」手順で装置駆動の危険を非殺で処理

7. 第四章の作成（01〜05）
   - 照合（スィグナ）と衝突（トリーネ）→夜の牽制→評価の収束→小さな和解
   - 正規様式と現場速度のズレを短いやり取りで提示

8. 第五章の作成（01〜05）
   - 露払い→首謀の影は名前を出さず示唆→非殺の成果→配分→巻の余韻
   - 乾いた締めの一言で余韻を残す

9. 目次と巻プロットの更新
   - docs/目次.txt に本文・プロットの全ファイルを列挙
   - docs/プロット/第一巻プロット.txt に各章のシーン構成・本文進捗を反映

10. .md → .txt への移行
   - docs/ 配下の .md をすべて同階層名で .txt に複製
   - 複製後、.md は削除（ソースは .txt に一本化）

11. VSCodeでの自然ソート用リネーム
   - 本文: 00_プロローグ.txt、nn-mm_タイトル.txt（nn=章、mm=シーン）
   - プロット: 00_プロローグ_構成.txt、nn-mm_タイトル_構成.txt

12. コミット運用（参考）
   - 例1: toneガイド追加・口調調整「tone: add Nork tone guide and viewpoint rules; refactor dialogue…」
   - 例2: 第二章以降の追加＋txt複製「draft: add chapters 2–5 (plots + texts); add .txt copies…」
   - 例3: .md削除「chore: remove Markdown sources under docs (migrated to .txt)」
   - 例4: 並び替えリネーム「chore: rename docs to numeric-prefixed filenames for VSCode sorting」

命名・文体・視点の規則（要約）
- 命名: 本文＝nn-mm_タイトル.txt、プロット＝nn-mm_タイトル_構成.txt、プロローグは 00_
- ノルクの台詞: 簡潔＋最小の配慮。必要時のみ皮肉を一言。言い捨て連発は避ける。
- 視点・叙述: ノルクの観察・推定に基づく。観測根拠が明確な差（技量・出力・数値）は断定可。血統・内面・動機は示唆に留める。

再現のチェックポイント

- 各章5シーンの本文とプロットが対応していること
- 目次と巻プロットに全エントリが載っていること
- .txt に一本化され、ファイル名が数値プレフィックスで揃っていること
