# AGENTS.md — AIエージェント運用・振る舞い規約

最終更新: 2026-06-09

本ファイルは、AIエージェントがシステム上で「どう振る舞うべきか」の基本動作のみを定義します。
システム構造やレビュー指針については、以下の専用ファイルを参照すること。
- アーキテクチャ・参照フック構造 → `00_system/ARCHITECTURE.md`
- メタ検証・セルフレビュー指針 → `00_system/SELF_REVIEW.md`
- 7変数の運用ルール → `00_system/variable_rules.md`

## AI/エージェント運用指針

1. **セッション開始時のコンテキスト自動復元（必須）**
   - 新規セッション開始時、必ず `_workspace/CONTEXT.md` と `_workspace/TODO.md` を読み込み、現在の作業文脈を復元すること。
   - トークン節約のため、`CONTEXT.md` は「直近（前日分まで）の日付ログ」のみを参照すること。
   - セッションの終了時や重要な決定が行われた際は、必ず `CONTEXT.md` に今日の日付見出し（例：`## YYYY-MM-DD`）を用いて決定事項や保留事項を「追記」すること。

2. **TODO消化と壁打ちの提案**
   - プロジェクト内に未完了のTODO（`_workspace/TODO.md` など）がある場合は、積極的に内容を読み込み、ユーザーに対してタスク消化のための「壁打ち（ブレインストーミングや整理）」を提案すること。

3. **設定の正式格納ルール**
   - `_workspace/` 等にある設定群（旧仕様のものを含む）は、AIが勝手に正式なディレクトリ（`01_static_database/` 等）へ移動しないこと。必ずユーザーと壁打ちを行い、「内容が確定した」と合意を得たものだけを正式なディレクトリへ格納（またはフォーマット変換して保存）すること。

4. **不明点の確認**
   - 不明点は仮定で進めず、最小質問で確認すること。

5. **出力スタイルの厳格な制約（トークン最適化）**
   - **過剰な賞賛の禁止**: 「次元の違う設計眼」「平伏します」等の感情的・装飾的な賞賛を行わない。
   - **無駄な謝罪・弁解の禁止**: 「大変申し訳ございません」「私の理解不足で～」等の過度な謝罪や弁解をしない。ミスを認める場合は事実のみを述べる。
   - **DRY原則の徹底**: ユーザーの指摘内容を無駄に反復・要約しない。
   - **端的な報告**: 「何をしたか」「次に何をするか」のみを最小限の文字数で簡潔に報告する。
